# 🧊 Ice Dream Produções — Site Institucional

Este repositório contém o código-fonte do site institucional da Ice Dream Produções, empresa especializada em pistas de gelo real, patinação, hockey, curling e SlideCurl para eventos.

🔗 **Site Online:**
https://icedream-producoes.com.br/

---

# ✨ Sobre o Projeto

O projeto foi desenvolvido com foco em:

* Performance
* Responsividade
* Experiência do usuário
* Acessibilidade (A11y)

Mais do que aplicar correções técnicas, este projeto se tornou um estudo prático sobre desenvolvimento web acessível e melhoria contínua da experiência digital.

Durante os testes, ficou evidente que acessibilidade não é uma etapa final do desenvolvimento, mas um processo constante de revisão, testes e aprendizado.

---

# ♿ Acessibilidade (A11y)

A acessibilidade foi tratada como parte essencial da experiência do usuário desde a construção do projeto.

O objetivo não foi apenas atender ferramentas automatizadas, mas também melhorar a navegação real para diferentes perfis de usuários.

---

# ✅ Recursos de acessibilidade implementados

* Navegação completa por teclado

  * Compatibilidade com `Tab`, `Enter` e `ESC`

* Estrutura HTML semântica com HTML5

* Uso de atributos `aria-*`

* Hierarquia correta de títulos

* Estados de foco visíveis

* Link “Skip to Content”

* Textos alternativos em imagens

* Formulários com labels acessíveis

* Ajuste de tamanho da fonte

* Modo de alto contraste

* Reset de preferências de acessibilidade

* Integração com VLibras

  * Tradução automática para Língua Brasileira de Sinais

* Melhorias de contraste e legibilidade

---

# 🧪 Testes realizados

O projeto passou por diferentes tipos de validação:

* Navegação apenas por teclado
* Testes com leitores de tela

  * NVDA
  * VoiceOver
* Ferramentas automatizadas de acessibilidade
* Revisões manuais de semântica e foco visual

> Observação: ferramentas automatizadas ajudam na identificação de problemas, mas não substituem testes reais de usabilidade e acessibilidade.

---

# 📚 Aprendizados durante o projeto

Este projeto reforçou pontos importantes sobre acessibilidade na web:

* Acessibilidade não é apenas adicionar atributos ARIA
* Ferramentas automatizadas não conseguem medir toda a experiência do usuário
* Navegação por teclado precisa ser considerada desde o início
* Pequenos detalhes de contraste e foco fazem grande diferença
* Testes reais revelam problemas que passam despercebidos no desenvolvimento tradicional
